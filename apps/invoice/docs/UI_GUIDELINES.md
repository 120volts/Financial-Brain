# UI Guidelines

## Character

Professional, clean and practical. The interface should resemble trustworthy business software without becoming dense or bureaucratic.

## Interaction

- Large touch targets on iPhone
- Clear selected state in bottom navigation
- Dashboard cards navigate to the corresponding filtered list
- Every control that appears pressable must have a real action
- Destructive actions require confirmation and should support undo when feasible
- Forms should minimize repeated entry

## Status colors

- Draft: yellow
- Overdue: red
- Paid: green
- Other statuses: distinct but less alarming

Color is supplemental; text labels remain required for accessibility.

## Responsive behavior

- Phone: rapid invoice capture and status updates
- Desktop: denser lists, detailed editing and review
- Core actions must remain reachable without horizontal scrolling
